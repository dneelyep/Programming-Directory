G.b.syncengine : module documentation

Part of GTG.backends

This library deals with synchronizing two sets of objects. It works like this:

  • We have two sets of generic objects (local and remote)
  • We present one object of either one of the sets and ask the library what's the state of its synchronization
  • the library will tell us if we need to add a clone object in the other set, update it or, if the other one has been removed, remove also this one
Class SyncMeme A SyncMeme is the object storing the data needed to keep track of the state
Class SyncMemes A TwoKeyDict, with just the names changed to be better understandable.
Class SyncEngine The SyncEngine is an object useful in keeping two sets of objects
